**Callback**

Antes de las promesas, las operaciones as√≠ncronas se manejaban con callbacks, lo que generaba el problema de los callbacks hell, que es la anidaci√≥n de callbacks, lo que dificulta la lectura y mantenimiento del c√≥digo.

Para simular la asincron√≠a, se utiliza la funci√≥n `setTimeout`, que simula que la petici√≥n tarda 2 segundos en responder.

```js

  console.log('--- Inicio de la petici√≥n ----');

  setTimeout(() => {
    
    const users = [
      { id: 1, name: 'Juan' },
      { id: 2, name: 'Pedro' },
      { id: 3, name: 'Mar√≠a' }
    ];

    console.log(' ::: Petici√≥n resuelta :::');
    callback(users);  //llamamos a la funci√≥n de vuelta (callback)
```

Ahora para llamara a la funci√≥n getUsers, se hace de la siguiente forma:


El problema de los callballs, es el llamado `callback hell`, que es la anidaci√≥n de callbacks, lo que dificulta la lectura y mantenimiento del c√≥digo.


### Promesas 4.2

**Promesas**

> üí° Las promesas son objetos que representan la terminaci√≥n o el fracaso de una operaci√≥n as√≠ncrona.


Siguiente el ejemplo anterior, vamos a refactorizar el c√≥digo para utilizar promesas.

```js
function getUsers() {
  
  console.log('--- Inicio de la petici√≥n ----');

  return new Promise((resolve, reject) => {
    setTimeout(() => {
      
      const users = [
        { id: 1, name: 'Juan' },
        { id: 2, name: 'Pedro' },
        { id: 3, name: 'Mar√≠a' }
      ];

      console.log(' ::: Petici√≥n resuelta :::');
      resolve(users);

    }, 2000);
  });
  
  console.log(' --- Fin de la petici√≥n ---');
}
```	

En el caso de las promesas, la funci√≥n `resolve` es el equivalente al callback, y se llama cuando la operaci√≥n as√≠ncrona se ha completado.

Ahora para llamar a la funci√≥n getUsers, se hace de la siguiente forma:

```js
getUsers()
  .then((users) => {
          console.log(users);
        })

  .catch((error) => {
          console.log(error);
        });
```

En la parte de `then` se recibe el resultado de la promesa, y en `catch` se recibe el error en caso de que la promesa falle.


**Encadena promesas**

En el caso de las promesas, se pueden encadenar, para evitar el callback hell.

```js
getUsers()
  .then((users) => {
    return getRoles(users);
  })
  .then((roles) => {
    return getPermissions(roles);
  })
  .then((permissions) => {
    console.log(permissions);
  })
  .catch((error) => {
    console.log(error);
  });
```
De esta forma se evita el callback hell, y se mantiene un c√≥digo m√°s limpio y f√°cil de leer. En cuanto un error se produzca, se captura en el `catch`.



### Async/Await 4.3

üÜò Ayuda: [Async/Await](https://playcode.io/javascript/async-await)

> üí° Async/Await es una forma de trabajar con promesas de forma m√°s sencilla y legible, es como escribir c√≥digo as√≠ncrono de la misma forma que escribimos c√≥digo s√≠ncrono (la forma tradicional)

```js
        { id: 3, name: 'Mar√≠a' }
```


Si se desea exportar varias funciones, variables o constantes, se puede hacer de la siguiente forma:

```js

  return a - b;

export let name = 'Juan';
```

Y para importar varias funciones, variables o constantes, se puede hacer de la siguiente forma:

```js
//archivo: index.js
```

**Exportar e Importar por defecto**

Para exportar por defecto una funci√≥n, variable o constante, se utiliza la palabra clave `export default`.

```js
//archivo: utils.js
export default function sum(a, b) {
  return a + b;
}
```

Para importar por defecto la funci√≥n, variable o constante, se utiliza la palabra clave `import`, y se puede asignar a cualquier nombre.

```js

```

### Eventos del DOM 5.2

üÜò Ayuda: [Eventos DOM en JavaScript](https://developer.mozilla.org/en-US/docs/Learn/JavaScript/Building_blocks/Events)

> üí° Los eventos son acciones que se producen en el DOM, como hacer click, pasar el mouse, escribir, etc. Cuando se quiere responder ante un evento, es necesario adjuntar lo que se llama un `listener` o manejador de eventos.

**Eventos del DOM**

Antes de trabajar con eventos, es necesario seleccionar el elemento al que se le va a adjuntar el evento. Para ello tenemos diferentes opciones, todas ellas a trav√©s del objeto `document`.

- `document.getElementById('id')`: Selecciona un elemento por su id.
- `document.querySelector('selector')`: Selecciona un elemento por un selector CSS. Devuelve el primer elemento que cumpla con el selector, o `null` si no encuentra ning√∫n elemento.
- `document.querySelectorAll('selector')`: Selecciona todos los elementos que cumplan con el selector CSS. Devuelve un NodeList.


El selector CSS puede ser una clase, un id, un tag, un atributo, etc. Por ejemplo: `#id`, `.class`, `tag`, `[atributo]`, etc. Para indicar esto es `querySelector` se utiliza el mismo formato que en CSS, por ejemplo: `#id`, `.class`, `tag`, `[atributo]`, etc.


```js
//Seleccionar un elemento por su id
const element = document.getElementById('id');

//Seleccionar un elemento por un selector CSS
const element = document.querySelector('.class');

//Seleccionar todos los elementos que cumplan con el selector CSS

```

**Adjuntar un evento**

Para adjuntar un evento a un elemento, se utiliza el m√©todo `addEventListener`, que recibe dos par√°metros, el nombre del evento y la funci√≥n que se ejecutar√° cuando se produzca el evento.

> üí° La lista de eventos es muy extensa aqu√≠ est√° la [lista completa de eventos.](https://developer.mozilla.org/es/docs/Web/Events)

```js

  console.log('Click en el elemento');
});
```

Lo normal es indicar la funci√≥n cmo un arrow function o una funci√≥n an√≥nima, pero tambi√©n se puede indicar una funci√≥n con nombre.

```js

  console.log('Click en el elemento');
}
element.addEventListener('click', handleClick);

//funci√≥n an√≥nima

  console.log('Click en el elemento');
});

//Arrow function

  console.log('Click en el elemento');
});
```

Si tenemos varios elementos, se puede recorrer el NodeList con un bucle, pero lo mejor es utilizar la un m√©todo `forEach`. Para ello es necesario convertir el NodeList a un array. `Array.from()` o el operador spread `...`.

Por ejemplo:



```js

//Con Array.from()

  element.addEventListener('click', function() {
    console.log('Click en el elemento');
  });
});


//Con el operador spread, podemos convertir un NodeList a un array

  element.addEventListener('click', function() {
    console.log('Click en el elemento');
  });
});

//Tambien se puede utilizar la misma functi√≥n para todos los elementos

  console.log('Click en el elemento');
}

  element.addEventListener('click', handleClick);
});
```
